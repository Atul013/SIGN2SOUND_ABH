FRAMEWORK
PyTorch 2.0+ with CUDA support
Rationale: Dynamic graphs, efficient RNN/LSTM/GRU, variable-length sequences, edge deployment (TorchScript, ONNX)

ARCHITECTURE: BIDIRECTIONAL GRU
Type: Hybrid RNN with Dense Layers
Suitability: Temporal modeling, 447K params (efficient), bidirectional context, edge-friendly (1.7MB, <10ms inference)

Layer Structure:
INPUT (32, 1, 63) → Bi-GRU (63→128 hidden, 2 layers, dropout=0.3) → (32, 1, 256)
→ Last timestep → (32, 256) → FC1 (256→256) + ReLU + Dropout(0.3)
→ FC2 (256→128) + ReLU + Dropout(0.3) → Output (128→29)

Parameters: GRU=344,832 (75.4%) | FC1=65,792 (14.4%) | FC2=32,896 (7.2%) | Output=3,741 (0.8%)
TOTAL: 447,261 params | Size: 1.7MB (FP32), 0.85MB (FP16), 0.43MB (INT8)

TRANSFER LEARNING
MediaPipe Hand Landmarker: Frozen feature extractor, pre-trained on 30K+ images, >95% detection
GRU: Trained from scratch (landmark features already high-level, 223K samples sufficient)

CUSTOM COMPONENTS
• Wrist-Relative Normalization: landmark_norm = (landmark - wrist) / bbox_diagonal
• Focal Loss: α=0.25, γ=2.0 (class imbalance)
• Bidirectional GRU on single frames: 2x features, +3-5% accuracy
• Hybrid Dense: 256→256→128 (dimensionality preservation before compression)
